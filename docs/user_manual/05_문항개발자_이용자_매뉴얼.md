## 2. 대시보드

> **[이미지 필요: IMG-R-01]** 문항개발자 대시보드 전체 — 통계 카드 + 빠른 액션 + 최근 활동

### 2.1 통계 카드 (4개)

| 카드 | 내용 | 클릭 시 이동 |
|------|------|-------------|
| **전체 문항** | 등록된 총 문항 수 | 문항 관리 |
| **완성 문항** | 지문이 연결된 문항 수 | 모듈 관리 |
| **활성 문항** | 활성 상태 문항 수 | 문항 관리 (활성 필터) |
| **전체 지문** | 등록된 총 지문 수 | 지문 관리 |

### 2.2 빠른 액션 카드 (4개)

- PDF 업로드 (모듈 자동 생성)
- 새 문항 등록
- 새 지문 등록
- 진단지 구성

### 2.3 최근 활동

- 최근 등록된 문항 5개
- 최근 등록된 지문 5개

## 3. 평가 영역 관리

**메뉴 위치:** 사이드바 > **평가 영역**

> **[이미지 필요: IMG-R-02]** 평가 영역 관리 화면 — 영역 목록 + 세부 지표

### 3.1 평가 영역이란?

평가 영역은 학생의 읽기 능력을 측정하는 **대분류 기준**입니다. 각 영역 아래에 **세부 지표**(소분류)가 있습니다.

```
평가 영역 (EvaluationIndicator)
  └── 세부 지표 1 (SubIndicator)
  └── 세부 지표 2 (SubIndicator)
  └── ...
```

### 3.2 평가 영역 관리

| 기능 | 설명 |
|------|------|
| **영역 추가** | 새 평가 영역 생성 (AJAX 즉시 반영) |
| **세부 지표 추가** | 영역 하위에 세부 지표 생성 |
| **통계 확인** | 각 영역/지표에 연결된 문항 수 확인 |

### 3.3 통계 카드

| 통계 | 내용 |
|------|------|
| 전체 영역 수 | 등록된 평가 영역 총 수 |
| 활성 영역 수 | 활성 상태인 영역 수 |
| 검토 중 | 검토가 필요한 영역 수 |
| 영역당 평균 지표 수 | 세부 지표 평균 개수 |

## 4. 모듈 관리 (문항 은행)

**메뉴 위치:** 사이드바 > **모듈 관리**

> **[이미지 필요: IMG-R-03]** 모듈 관리 카드 뷰 — 필터, 검색, 모듈 카드

### 4.1 모듈이란?

**모듈 = 1개 읽기 지문(ReadingStimulus) + 여러 문항(Items)**

완성된 평가 단위로, 진단지를 만들 때 이 모듈들을 조합합니다.

### 4.2 모듈 목록 보기

- **카드 형식**으로 모듈이 표시됩니다.
- 각 카드에 표시되는 정보:

| 항목 | 설명 |
|------|------|
| 지문 제목 | 읽기 지문의 제목 |
| 객관식 N개 | MCQ 문항 수 |
| 서술형 N개 | 주관식 문항 수 |
| 전체 N개 | 총 문항 수 |
| 핵심 개념 | 배지로 표시되는 핵심 키워드 |
| 예상 소요 시간 | 객관식×2분 + 서술형×5분 |
| 상태 | draft / active / archived |

### 4.3 필터 및 검색

| 필터 | 옵션 |
|------|------|
| **검색** | 지문 제목, 코드, 내용으로 검색 |
| **상태** | 전체 / 작업중(draft) / 배포가능(active) / 보관됨(archived) |

### 4.4 모듈 상세 보기

카드를 클릭하면 지문 전체 내용과 연결된 문항 목록을 확인할 수 있습니다.

> **[이미지 필요: IMG-R-04]** 모듈 상세 — 지문 내용 + 문항 목록 + 편집 버튼

### 4.5 모듈 관리 기능

| 기능 | 설명 |
|------|------|
| **AI 분석** | 지문의 구조와 핵심 개념을 AI가 분석 |
| **복제** | 모듈 전체를 복사하여 새 모듈 생성 |
| **보관** | 모듈을 archived 상태로 변경 (목록에서 숨김) |
| **복구** | 보관된 모듈을 다시 활성화 |
| **정답지 업로드** | PDF 형태의 정답지 업로드 |
| **정답 일괄 수정** | CSV로 정답 데이터 일괄 변경 |

## 5. 모듈 자동 생성

**메뉴 위치:** 사이드바 > **모듈 자동 생성**

> **[이미지 필요: IMG-R-05]** 모듈 자동 생성 화면 — 후보 목록, 승인/거절 버튼

### 5.1 PDF 업로드를 통한 자동 생성

1. 대시보드 또는 지문 관리에서 **"PDF 업로드"** 버튼을 클릭합니다.
2. 읽기 자료 PDF 파일을 선택합니다.
3. AI(GPT-4)가 **백그라운드에서** PDF를 분석합니다:
   - 지문 추출
   - 객관식 문항 추출 (선택지 포함)
   - 서술형 문항 추출
4. "파싱 중..." 상태가 표시됩니다.
5. 완료 후 모듈 자동 생성 목록에서 결과를 확인합니다.

> **[이미지 필요: IMG-R-06]** PDF 업로드 진행 화면 — 파싱 중 스피너

### 5.2 생성 후보 검토

AI가 생성한 모듈 후보를 검토합니다:

| 작업 | 설명 |
|------|------|
| **승인** | 모듈을 정식 등록 |
| **거절** | 모듈 후보 삭제 |
| **재생성** | AI에게 다시 생성 요청 |

## 6. 지문 관리

**메뉴 위치:** 사이드바 > **지문 관리**

> **[이미지 필요: IMG-R-07]** 지문 관리 목록 — 검색, 필터, CRUD 버튼

### 6.1 지문 목록

| 표시 정보 | 설명 |
|-----------|------|
| 제목 | 지문 제목 |
| 문항 수 | 연결된 문항 수 (counter cache) |
| 상태 | draft / active / archived |
| 생성일 | 등록 날짜 |

### 6.2 새 지문 등록

1. **"새 지문 등록"** 버튼을 클릭합니다.
2. 다음 정보를 입력합니다:

| 필드 | 설명 | 필수 |
|------|------|------|
| 제목 | 지문의 제목 | O |
| 본문 | 읽기 지문 내용 | O |
| 출처 | 원문 출처 | X |
| 장르 | 문학/비문학 등 | X |
| 난이도 | 쉬움/보통/어려움 | X |

3. **"저장"** 버튼을 클릭합니다.

> **[이미지 필요: IMG-R-08]** 지문 등록 폼

### 6.3 지문 편집 / 삭제

- **편집**: 지문 내용 수정
- **삭제**: 지문과 연결된 **모든 문항, 루브릭, 응답 데이터가 함께 삭제**됩니다.
  - 삭제 전 확인 대화상자가 표시됩니다.

### 6.4 정답지 관리

| 기능 | 설명 |
|------|------|
| **정답지 업로드** | PDF 형태의 정답/해설 업로드 |
| **CSV 템플릿 다운로드** | 정답 입력용 CSV 템플릿 |
| **CSV 업로드** | 정답 데이터 일괄 등록 |
| **정답 일괄 수정** | 기존 정답 데이터 변경 |

## 7. 문항 관리 / 문항 등록

### 7.1 문항 관리

**메뉴 위치:** 사이드바 > **문항 관리**

> **[이미지 필요: IMG-R-09]** 문항 관리 테이블 뷰 — 필터, 검색, 삭제

| 열 | 내용 |
|----|------|
| 코드 | 문항 고유 코드 |
| 제목/프롬프트 | 문항 내용 요약 |
| 유형 | MCQ / 서술형 |
| 난이도 | 쉬움 / 보통 / 어려움 |
| 지문 | 연결된 지문명 |
| 상태 | draft / active / archived |
| 생성일 | 등록 날짜 |

**필터 옵션:**
- 상태별 (전체/draft/active/archived)
- 루브릭 유무 (서술형 채점 기준)
- 키워드 검색 (코드/프롬프트)

### 7.2 문항 등록

**메뉴 위치:** 사이드바 > **문항 등록**

> **[이미지 필요: IMG-R-10]** 문항 등록 폼 — 전체 필드

1. **"새 문항 추가"** 또는 사이드바 **"문항 등록"**을 클릭합니다.
2. 다음 정보를 입력합니다:

| 필드 | 설명 | 필수 |
|------|------|------|
| 코드 | 문항 고유 코드 | O |
| 유형 | MCQ / 서술형 | O |
| 난이도 | 쉬움/보통/어려움 | O |
| 평가 영역 | EvaluationIndicator 선택 | X |
| 세부 지표 | SubIndicator 선택 | X |
| 프롬프트 | 문항 내용 (질문) | O |
| 해설 | 정답 해설 | X |
| 지문 | 연결할 ReadingStimulus 선택 | X |
| 상태 | draft / active | O |

3. **"저장"** 버튼을 클릭하면 **편집 페이지**로 이동합니다.

### 7.3 문항 편집 (정답/루브릭 설정)

저장 후 편집 페이지에서 추가 설정을 합니다:

|-----------|------|

| **기준(Criterion) 추가** | 채점 기준 항목 추가 |
| **수준(Level) 설정** | 각 기준별 점수 수준 설정 |

> **[이미지 필요: IMG-R-12]** 문항 편집 — 서술형 루브릭 설정 화면

## 8. 진단지 구성

**메뉴 위치:** 사이드바 > **진단지 구성**

> **[이미지 필요: IMG-R-13]** 진단지 목록 화면 — 상태별 필터

### 8.1 진단지란?

**진단지 = 여러 모듈을 조합한 완성된 평가지**

학생에게 실제로 배정되어 응시하는 단위입니다.

### 8.2 진단지 목록

| 표시 정보 | 설명 |
|-----------|------|
| 이름 | 진단지 이름 (예: "1학년 1학기 중간평가") |
| 포함 문항 수 | 자동 계산 |
| 상태 | draft / active |
| 생성일 | 등록 날짜 |

### 8.3 새 진단지 생성

1. **"새 진단지 만들기"** 버튼을 클릭합니다.
2. 기본 정보를 입력합니다:

| 필드 | 설명 |
|------|------|
| 이름 | 진단지 이름 |
| 설명 | 진단지 설명 |
| 제한 시간 | 응시 제한 시간 (분) |

3. **모듈 선택**: 체크박스로 포함할 모듈을 선택합니다.

> **[이미지 필요: IMG-R-14]** 진단지 생성 — 모듈 선택 체크박스 인터페이스

- active 상태인 모듈만 선택 가능
- 선택한 모듈이 파란색 테두리로 강조 표시
- 각 모듈의 문항 수, 소요 시간 확인 가능

4. **"생성하기"** 버튼을 클릭합니다.

### 8.4 진단지 편집

- draft 상태인 진단지만 편집 가능합니다.
- 모듈을 추가/제거하면 기존 연결이 초기화되고 새로 생성됩니다.

### 8.5 진단지 배포 / 배포 취소

| 작업 | 상태 변경 | 의미 |
|------|----------|------|
| **배포** | draft → active | 교사가 학생에게 배정 가능 |
| **배포 취소** | active → draft | 배정 불가 (기존 배정은 유지) |

### 8.6 진단지 삭제

- draft 상태에서만 삭제 가능합니다.
- active 상태 진단지를 삭제하려면 먼저 배포 취소해야 합니다.

## 12. 자주 묻는 질문 (FAQ)

### Q. PDF를 업로드했는데 모듈이 안 생겨요.
**A.** PDF 파싱은 백그라운드에서 진행됩니다. "모듈 자동 생성" 메뉴에서 결과를 확인하세요. 실패한 경우 재업로드해 주세요.

### Q. 모듈의 문항 수가 실제와 다르게 표시돼요.
**A.** 문항을 추가/삭제하면 자동으로 메타데이터가 재계산됩니다. 즉시 반영되지 않으면 모듈 상세 페이지를 새로고침하세요.

### Q. 진단지를 삭제하면 학생 데이터도 삭제되나요?
**A.** 아니요, 진단지를 삭제해도 이미 생성된 학생 응답(StudentAttempt, Response)은 유지됩니다. 다만 해당 진단지로의 신규 배정은 불가합니다.

### Q. 서술형 문항에 루브릭을 꼭 만들어야 하나요?
**A.** 루브릭이 없으면 AI 자동 채점이 불가하고 교사가 수동 채점해야 합니다. 정확한 평가를 위해 루브릭 설정을 권장합니다.

### Q. active 상태인 모듈을 수정해도 되나요?
**A.** 수정은 가능하지만, 이미 이 모듈이 포함된 진단지로 응시한 학생 데이터에는 영향이 없습니다. 새로 응시하는 학생부터 변경 사항이 적용됩니다.
