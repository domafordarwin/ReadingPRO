## 2. 시스템 관리 (모니터링)

**메뉴 위치:** 사이드바 > **시스템 관리**

> **[이미지 필요: IMG-A-01]** 시스템 관리 대시보드 — 성능 메트릭 + 트렌드 차트

### 2.1 실시간 성능 메트릭

최근 5분간의 성능 데이터를 실시간으로 표시합니다.

| 지표 | 설명 | 정상 기준 |
|------|------|----------|
| **FCP** (First Contentful Paint) | 첫 콘텐츠 표시 시간 | < 1.8초 |
| **LCP** (Largest Contentful Paint) | 가장 큰 요소 로딩 시간 | < 2.5초 |
| **CLS** (Cumulative Layout Shift) | 레이아웃 변동 | < 0.1 |
| **INP** (Interaction to Next Paint) | 상호작용 반응 시간 | < 200ms |
| **TTFB** (Time to First Byte) | 서버 응답 시간 | < 500ms |

### 2.2 24시간 트렌드

- 시간별로 성능 지표의 변화를 차트로 표시합니다.
- 피크 시간대와 성능 저하 구간을 파악할 수 있습니다.

> **[이미지 필요: IMG-A-02]** 24시간 트렌드 차트

### 2.3 학생 진단 데이터 (JSON)

- `/admin/system/student_diagnostics` 엔드포인트로 학생 진단 데이터를 JSON 형식으로 조회할 수 있습니다.
- 외부 분석 도구와 연동할 때 활용합니다.

## 3. 사용자 관리

**메뉴 위치:** 사이드바 > **사용자 관리**

> **[이미지 필요: IMG-A-03]** 사용자 관리 화면 — 목록, 검색, 역할 필터

### 3.1 사용자 목록

- **전체 사용자 목록**: 시스템에 등록된 모든 사용자
- **검색**: 이메일로 검색
- **역할 필터**: student / parent / teacher / diagnostic_teacher / school_admin / researcher / admin

| 표시 정보 | 설명 |
|-----------|------|
| 이메일 | 로그인 ID |
| 이름 | 사용자 이름 |
| 역할 | 현재 역할 |
| 상태 | 활성 / 잠금 |
| 마지막 로그인 | 최근 접속일 |

### 3.2 사용자 생성

1. **"새 사용자"** 버튼을 클릭합니다.
2. 다음 정보를 입력합니다:

| 필드 | 설명 | 필수 |
|------|------|------|
| 이메일 | 로그인 ID | O |
| 이름 | 사용자 이름 | O |
| 역할 | 7개 역할 중 선택 | O |
| 비밀번호 | 초기 비밀번호 | O |

3. **"생성"** 버튼을 클릭합니다.

### 3.3 역할 변경

1. 사용자 목록에서 대상 사용자를 찾습니다.
2. **"역할 변경"** 버튼을 클릭합니다.
3. 새 역할을 선택합니다.
4. **"변경"** 버튼을 클릭합니다.

> **[이미지 필요: IMG-A-04]** 역할 변경 인터페이스

**주의:** 역할을 변경하면 해당 사용자의 접근 권한이 즉시 변경됩니다.

### 3.4 비밀번호 재설정

1. 사용자 옆의 **"비밀번호 재설정"** 버튼을 클릭합니다.
2. 임시 비밀번호가 자동 생성됩니다.
3. 임시 비밀번호를 사용자에게 전달합니다.
4. 사용자는 다음 로그인 시 비밀번호 변경이 필요합니다.

## 4. 역할 관리

**메뉴 위치:** 사이드바 > **역할 관리**

> **[이미지 필요: IMG-A-05]** 역할 관리 화면 — 7개 역할 목록

시스템에 정의된 7개 역할을 **읽기 전용**으로 확인합니다.

| 역할 | 한국어명 | 주요 권한 |
|------|---------|----------|
| **student** | 학생 | 진단 응시, 결과 확인, 발문 학습 |
| **parent** | 학부모 | 자녀 결과 확인, 상담 신청 |
| **teacher** | 교사 | 피드백 관리, 발문 관리 |
| **diagnostic_teacher** | 진단교사 | 전체 진단 관리, 보고서 생성 |
| **school_admin** | 학교관리자 | 학교 내 학생/학부모/진단 관리 |
| **researcher** | 문항개발자 | 문항/지문/진단지/모듈 관리 |
| **admin** | 시스템관리자 | 전체 시스템 관리 |

## 6. 에러 로그

**메뉴 위치:** 사이드바 > **에러 로그**

> **[이미지 필요: IMG-A-07]** 에러 로그 목록 — 필터, AI 분석 버튼

### 6.1 에러 목록 조회

시스템에서 발생한 에러를 자동으로 수집하여 보여줍니다.

| 표시 정보 | 설명 |
|-----------|------|
| 에러 메시지 | 에러 내용 요약 |
| 발생 위치 | 컨트롤러/액션 |
| 발생 시각 | 에러 발생 시간 |
| 상태 | 미해결 / 해결됨 |

### 6.2 에러 상세 보기

- 에러를 클릭하면 상세 정보(스택 트레이스, 파라미터, 사용자 정보)를 확인할 수 있습니다.

### 6.3 AI 에러 분석

1. 에러 상세 페이지에서 **"AI 분석"** 버튼을 클릭합니다.
2. GPT-4가 에러 원인을 분석하고 해결 방안을 제시합니다.
3. 분석 결과를 참고하여 개발팀에 전달합니다.

> **[이미지 필요: IMG-A-08]** AI 에러 분석 결과 화면

### 6.4 에러 해결 표시

| 작업 | 방법 |
|------|------|
| **개별 해결** | 에러 상세에서 "해결됨" 버튼 클릭 |
| **일괄 해결** | 체크박스 선택 → "일괄 해결" 클릭 |

## 7. 자주 묻는 질문 (FAQ)

### Q. 사용자 비밀번호를 초기화하면 어떤 비밀번호로 설정되나요?
**A.** 시스템이 임시 비밀번호를 자동 생성합니다. 사용자에게 전달하고, 사용자가 다음 로그인 시 새 비밀번호를 설정하도록 안내하세요.

### Q. 역할을 잘못 변경했어요. 되돌릴 수 있나요?
**A.** 같은 방법으로 원래 역할로 다시 변경하면 됩니다. 역할 변경은 즉시 적용됩니다.

### Q. 에러 로그가 너무 많아요. 중요한 것만 보려면?
**A.** 상태 필터에서 "미해결"을 선택하면 아직 처리되지 않은 에러만 볼 수 있습니다. 해결된 에러는 "일괄 해결"로 정리하세요.

### Q. 시스템 성능이 느려요.
**A.** 시스템 관리 페이지에서 TTFB와 LCP 지표를 확인하세요. TTFB > 500ms이면 서버 리소스(메모리/CPU) 증가가 필요할 수 있습니다. Railway 대시보드에서 리소스를 확인하세요.

### Q. 새 학교를 등록하려면 어떻게 하나요?
**A.** 현재 학교 등록은 시드 데이터 또는 Rails 콘솔을 통해 진행합니다. 관리 UI에서의 학교 생성 기능은 추후 업데이트 예정입니다.
