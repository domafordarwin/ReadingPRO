## 3. 진단 관리

### 3.1 학교 담당자 관리

**메뉴 위치:** 사이드바 > 진단 관리 > **학교 담당자 관리**

학교별 관리자(school_admin)와 교사 목록을 확인합니다.

> **[이미지 필요: IMG-T-03]** 학교 담당자 관리 화면 — 학교별 담당자 목록

| 기능 | 설명 |
|------|------|
| 학교별 담당자 조회 | school_admin, teacher 목록 |
| 학교별 학생 현황 | 해당 학교 학생 수 확인 |

---

### 3.2 학생별 진단 배정

**메뉴 위치:** 사이드바 > 진단 관리 > **학생별 진단 배정**

학생에게 진단 검사를 배정하는 핵심 기능입니다.

> **[이미지 필요: IMG-T-04]** 진단 배정 목록 화면 — 필터, 배정 버튼 위치

#### 배정 목록 확인

- 상태별 필터: **활성** / **완료** / **취소**
- 페이지네이션: 20명씩 표시
- 검색 기능 제공

#### 새 진단 배정하기

1. **"진단 배정"** 버튼을 클릭합니다.
2. **진단지 선택**: 활성 상태인 진단지 중 하나를 선택합니다.
3. **학생 선택**: 배정할 학생을 선택합니다.
4. **마감일 설정** (선택): 응시 마감일을 지정합니다.
5. **"배정하기"** 버튼을 클릭합니다.

> **[이미지 필요: IMG-T-05]** 진단 배정 생성 폼 — 진단지 선택, 학생 선택, 마감일

#### 배정 취소

- 배정 목록에서 **"취소"** 버튼을 클릭합니다.
- **주의:** "진행 중" 또는 "배정됨" 상태만 취소 가능합니다.

#### 재배정

- 완료된 진단을 재응시하게 하려면 **"재배정"** 버튼을 클릭합니다.
- 새로운 시도(attempt)가 생성됩니다.

---

### 3.3 학생답 관리

**메뉴 위치:** 사이드바 > 진단 관리 > **학생답 관리**

학생 응답 데이터를 엑셀로 관리하는 기능입니다.

> **[이미지 필요: IMG-T-06]** 학생답 관리 화면 — 진단지별 목록, 업로드/다운로드 버튼

#### 템플릿 다운로드

1. 진단지를 선택합니다.
2. **"템플릿 다운로드"** 버튼을 클릭합니다.
3. 엑셀 파일(.xlsx)이 다운로드됩니다.
4. 엑셀에 학생 응답 데이터를 입력합니다.

#### 응답 업로드

1. 작성된 엑셀 파일을 선택합니다.
2. **"업로드"** 버튼을 클릭합니다.
3. 업로드 결과가 표시됩니다:
   - 처리된 학생 수
   - 생성된 시도 수
   - 생성된 응답 수
   - 자동 채점된 객관식 수
   - 건너뛴 항목
   - 오류 (있을 경우)

> **[이미지 필요: IMG-T-07]** 업로드 결과 표시 화면

#### 피드백 일괄 생성

1. 진단지의 **"피드백 생성"** 버튼을 클릭합니다.
2. AI가 백그라운드에서 모든 학생의 피드백을 자동 생성합니다.
3. 진행 상태가 표시됩니다: "생성 중..." → "완료"

---

### 3.4 진단 결과 리포트

**메뉴 위치:** 사이드바 > 진단 관리 > **진단 결과 리포트**

모든 학생의 진단 결과 리포트를 조회합니다.

- 학생별 리포트 목록 (검색 가능)
- 평균 점수 표시
- 10개씩 페이지네이션

## 4. 진단 분석

### 4.1 응시/채점 현황

**메뉴 위치:** 사이드바 > 진단 분석 > **응시/채점 현황**

> **[이미지 필요: IMG-T-08]** 응시/채점 현황 화면 — 통계 카드 + 학생별 목록

| 통계 | 내용 |
|------|------|
| **진행 중** | 현재 응시 중인 학생 수 |
| **완료** | 응시를 마친 학생 수 |
| **채점 대기** | 서술형 채점이 필요한 학생 수 |

- 학생별 응시 상태를 확인합니다.
- 20명씩 페이지네이션됩니다.

---

### 4.2 학생별 피드백

**메뉴 위치:** 사이드바 > 진단 분석 > **학생별 피드백**

> **[이미지 필요: IMG-T-09]** 학생별 피드백 목록 화면

#### 피드백 목록

- 학생별로 그룹화된 피드백 목록이 표시됩니다.
- 이미 배포 완료된 학생은 목록에서 제외됩니다.
- 검색 및 정렬 기능을 제공합니다.

#### 피드백 생성 (AI 자동)

학생별 피드백 상세 페이지에서:

| 버튼 | 기능 |
|------|------|
| **MCQ 피드백 생성** | 객관식 응답 기반 AI 피드백 생성 |
| **서술형 피드백 생성** | 서술형 응답 기반 AI 피드백 생성 |
| **전체 피드백 생성** | 객관식 + 서술형 한번에 생성 |
| **종합 피드백 생성** | 전체를 종합한 종합 피드백 생성 |

> **[이미지 필요: IMG-T-10]** 학생별 피드백 상세 — AI 피드백 생성 버튼들

#### 피드백 수정

- AI가 생성한 피드백을 확인하고 **직접 수정**할 수 있습니다.
- 수정 후 **"저장"** 버튼을 클릭합니다.

#### 피드백 배포

| 작업 | 방법 |
|------|------|
| **개별 배포** | 학생별 "배포" 버튼 클릭 |
| **일괄 배포** | 체크박스 선택 → "일괄 배포" 클릭 |
| **배포 취소** | "배포 취소" 버튼으로 되돌리기 |

- 배포된 피드백은 학생과 학부모에게 공개됩니다.

---

### 4.3 종합 결과 보고서

**메뉴 위치:** 사이드바 > 진단 분석 > **종합 결과 보고서**

> **[이미지 필요: IMG-T-11]** 종합 결과 보고서 목록 화면 — 상태 필터, 생성 버튼

**중요:** 피드백이 배포된 학생만 이 목록에 표시됩니다.

#### 보고서 상태

| 상태 | 의미 | 할 수 있는 작업 |
|------|------|----------------|
| **없음** | 보고서 미생성 | 생성 가능 |
| **초안** (draft) | 보고서 생성됨 (비공개) | 편집, 재생성, 배포 가능 |
| **배포됨** (published) | 학생/학부모에게 공개됨 | 배포 취소, 재생성 가능 |

#### 보고서 생성하기

1. 학생 옆의 **"보고서 생성"** 버튼을 클릭합니다.
2. 보고서 생성 페이지로 이동합니다.
3. 학생 정보, 진단 데이터, 피드백 요약을 확인합니다.
4. **"종합 보고서 생성"** 버튼을 클릭합니다.

> **[이미지 필요: IMG-T-12]** 보고서 생성 페이지 — 학생 정보 + 생성 버튼

5. AI가 **백그라운드에서** 7개 섹션의 보고서를 자동 생성합니다.
6. **"AI가 종합 보고서를 생성하고 있습니다..."** 메시지가 표시됩니다.
7. 완료되면 자동으로 보고서 페이지로 이동합니다.

> **[이미지 필요: IMG-T-13]** 보고서 생성 중 폴링 UI — 스피너 + 진행 메시지

#### 보고서 편집

생성된 보고서의 각 섹션을 개별적으로 편집할 수 있습니다:

1. 수정할 섹션의 **"편집"** 아이콘을 클릭합니다.
2. 리치 텍스트 에디터에서 내용을 수정합니다.
3. **"저장"** 버튼을 클릭합니다.

특정 섹션만 AI로 다시 생성하고 싶으면 **"재생성"** 버튼을 클릭합니다.

#### 보고서 배포/취소

| 작업 | 방법 | 결과 |
|------|------|------|
| **개별 배포** | "배포" 버튼 클릭 | 해당 학생/학부모에게 공개 |
| **일괄 배포** | 체크박스 → "일괄 배포" | 선택한 모든 학생에게 공개 |
| **일괄 생성** | "일괄 생성" 버튼 | 미생성 학생 전체 보고서 생성 |
| **배포 취소** | "배포 취소" 버튼 | 비공개로 전환 |

## 5. 발문 학습 관리

### 5.1 발문 모듈 관리

**메뉴 위치:** 사이드바 > 발문 학습 > **발문 모듈 관리**

> **[이미지 필요: IMG-T-14]** 발문 모듈 목록 화면 — 필터, 생성 버튼

#### 모듈 목록

- 수준별 필터: elementary_low / elementary_high / middle / high
- 상태별 필터: 활성 / 비활성
- 검색 기능 제공
- 각 모듈의 세션 통계 표시

#### 모듈 생성

1. **"새 모듈 만들기"** 버튼을 클릭합니다.
2. 다음 정보를 입력합니다:

| 항목 | 설명 |
|------|------|
| 모듈명 | 발문 학습 모듈의 이름 |
| 읽기 지문 | 연결할 ReadingStimulus 선택 |
| 수준 | elementary_low / elementary_high / middle / high |
| 발문 유형 | 단계별 발문 설정 |
| 상태 | 활성 / 비활성 |

3. **"저장"** 버튼을 클릭합니다.

> **[이미지 필요: IMG-T-15]** 발문 모듈 생성 폼

#### 모듈별 세션 확인

- 모듈 상세 페이지에서 **"세션 목록"** 탭을 클릭하면 해당 모듈을 학습한 학생 세션 목록을 볼 수 있습니다.

---

### 5.2 발문 배정 관리

**메뉴 위치:** 사이드바 > 발문 학습 > **발문 배정 관리**

> **[이미지 필요: IMG-T-16]** 발문 배정 관리 화면 — 배정 목록 + 버튼

#### 배정 생성

1. **"배정하기"** 버튼을 클릭합니다.
2. 발문 모듈을 선택합니다.
3. 배정할 학생을 선택합니다.
4. **"배정"** 버튼을 클릭합니다.

#### 일괄 배정

- **"일괄 배정"** 버튼으로 여러 학생에게 동시에 배정할 수 있습니다.

#### 배정 취소

- 상태별 필터에서 배정을 확인하고 **"취소"** 버튼을 클릭합니다.

---

### 5.3 학생 세션 현황

**메뉴 위치:** 사이드바 > 발문 학습 > **학생 세션 현황**

> **[이미지 필요: IMG-T-17]** 학생 세션 현황 — 통계 카드 + 세션 목록

| 통계 | 내용 |
|------|------|
| **총 세션** | 생성된 전체 발문 세션 수 |
| **활동 중** | 현재 학습 진행 중인 세션 |
| **완료** | 학습을 마친 세션 |

- 상태별 필터, 모듈별 필터 제공
- 20개씩 페이지네이션

#### 세션 상세 보기

세션을 클릭하면 학생의 학습 내용을 상세히 확인할 수 있습니다:

| 탭 | 내용 |
|------|------|
| **발문 (3단계)** | 학생이 작성한 질문과 AI 평가 결과 |
| **토론** | AI와의 대화 내용 |
| **에세이** | 학생이 작성한 논술문 |
| **보고서** | AI 생성 발문 역량 보고서 |

> **[이미지 필요: IMG-T-18]** 세션 상세 화면 — 3단계 발문 + 토론 + 에세이

#### 교사 리뷰 (채점)

1. 각 단계의 학생 발문에 **교사 점수**와 **피드백**을 입력합니다.
2. **"리뷰 완료"** 버튼을 클릭합니다.
3. 교사 점수가 반영되어 최종 점수가 재계산됩니다.

#### 단계별 피드백 배포

1. 특정 단계의 피드백을 학생에게 공개하려면:
2. 해당 단계에서 **"피드백 배포"** 버튼을 클릭합니다.
3. 해당 단계의 모든 발문에 교사 피드백이 배포됩니다.

#### 에세이 피드백

1. 에세이 탭에서 **교사 피드백**과 **점수**를 입력합니다.

## 7. 자주 묻는 질문 (FAQ)

### Q. 학생에게 진단을 배정했는데 학생이 목록에서 못 찾아요.
**A.** 학생이 로그인 후 대시보드 또는 "문해력 검사" 메뉴에서 배정된 진단을 확인할 수 있습니다. 마감일이 지난 경우에는 표시되지 않을 수 있습니다.

### Q. 보고서 생성이 실패했어요.
**A.** "보고서 생성 실패" 메시지가 표시되면 오류 내용을 확인하고, **"재생성"** 버튼으로 다시 시도해 보세요. 오류가 지속되면 관리자에게 문의하세요.

### Q. 보고서 생성 중에 다른 작업을 해도 되나요?
**A.** 네, 보고서는 백그라운드에서 자동 생성됩니다. 다른 페이지에서 작업하다가 나중에 확인하면 됩니다.

### Q. 피드백을 배포한 후 수정할 수 있나요?
**A.** "배포 취소" 후 수정하고 다시 배포할 수 있습니다.

### Q. 여러 학생의 보고서를 한번에 생성할 수 있나요?
**A.** 네, 종합 결과 보고서 목록에서 **"일괄 생성"** 버튼을 사용하면 모든 미생성 학생의 보고서를 한번에 생성할 수 있습니다.

### Q. 발문 학습에서 학생 수준은 어떻게 변경되나요?
**A.** 마스터리 점수 90% 이상 + 스캐폴딩 수준 0일 때 자동으로 다음 수준으로 올라갑니다. 수동 변경은 불가합니다.
